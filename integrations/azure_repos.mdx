---
title: "Azure Repos"
description: "Connect Refacto to your Azure DevOps Repos."
---

Follow these steps to connect Refacto to your Azure DevOps repositories.

<Steps>
  <Step title="Go to Integrations" icon="window">
    - Navigate to the **Integrations** page in your Refacto dashboard.
    - Click **Connect** on **Azure Repos**.

      ![Azure Integration Page Pn](/images/installation/azure/azure-integration-page.png)
  </Step>
  <Step title="Open Azure Repos Setup Form" icon="clipboard-list">
    - After clicking **Connect**, you’ll be redirected to the **Azure Repos setup form** in Refacto.
    - This form requires three fields: **PAT Token**, **Organisation Name**, and **Bot Email**.
    - To get these values, please follow the next steps:
      - Create a Review User
      - Generate a Personal Access Token (PAT)

    ![Azure Setup Form Empty Pn](/images/installation/azure/azure-form.png)
  </Step>
  <Step title="Create a Review User" icon="user">
    You need to create a dedicated **review user** in Azure DevOps.

    1. Navigate to **Organization Settings**.

       ![Org Setting Pn](/images/installation/azure/org-setting.png)
    2. Go to **Users** from the sidebar. Click **Add users**.

       ![Click On Add Users Pn](/images/installation/azure/click-on-add-users.png)
    3. Enter **email** or **review user**. Choose Access level as **Basic**. Choose the projects.

       ![Choose Project Pn](/images/installation/azure/choose-project.png)
    4. Choose **Project Adminstrator** as Azure Devops Group. It is required to create webhooks in **Service Hooks**.

       ![Project Admin Pn](/images/installation/azure/project-admin.png)
  </Step>
  <Step title="Generate a Personal Access Token (PAT)" icon="key">
    Azure DevOps provides an option to generate a **Personal Access token** for a user. Follow these steps to generate the token:

    1. Next to your avatar on the header’s right side is a settings icon, click it, then choose **Personal Access Tokens**.

       ![Pat1 Pn](/images/installation/azure/pat1.png)
    2. Click **New Token**.

       ![Pat2 Pn](/images/installation/azure/pat2.png)
    3. Enter a name and an expiry date for the token.

       ![Pat3 Pn](/images/installation/azure/pat3.png)
    4. Grant the following permissions:
       1. **Read & Write** Code.
       2. **Read** for Work items.
       3. **Read** Graph for user info like email, name, etc.

          ![Pat4 Pn](/images/installation/azure/pat4.png)

          ![Pat5 Pn](/images/installation/azure/pat5.png)

          ![Pat6 Pn](/images/installation/azure/pat6.png)

    <Note>
      Copy and save token. As its shown only once.
    </Note>
  </Step>
  <Step title="Fill in the Setup Form" icon="check">
    On the Azure Repos setup page in Refacto, provide the following:

    - **PAT Token** – Paste the token you generated above.
    - **Azure DevOps Organisation Name** – Extract this from your Azure DevOps URL.\
      Example:

      ```
      https://dev.azure.com/bluetowers/
      ```

      → organisation name is **bluetowers**.
    - **Bot Email** – Enter the email of the review user you created earlier.

    Click **Save Settings** when done.
  </Step>
  <Step title="Done" icon="party-horn">
    Your Azure Repos integration is now active.
  </Step>
</Steps>
